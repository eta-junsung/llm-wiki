# AM263P ADC 인스턴스 vs SOC(채널) 배치 결정 가이드라인

> **AM263P 플랫폼 설계 규칙 정본.** 신호를 어느 ADC 인스턴스/채널(SOC)에 배정할지 결정하는 기준. 형제 정본: [[am263p_adc_rti_trigger]](트리거 결선·EOC ISR), [[am263p_syscfg_soft_vs_hard_assign]](논리↔물리 배정 함정). 8kw 적용: [[adc_pinmap]].

## 전제 교정 (중요)

**여러 ADC 인스턴스를 쓰는 게 덜 안정적인 게 아니다.** AM263P 같은 모터·전력제어용 MCU가 ADC0~ADC4를 둔 이유 자체가 "동시에 여러 신호를 잡으라"는 것 — 멀티 인스턴스는 정상·표준 사용법이다. 진짜 차이는 **안정성이 아니라 ① 타이밍(동시성) ② 펌웨어 복잡도** 두 축이다.

## 핵심 하드웨어 사실 (아키텍처)

- **각 ADC 인스턴스(ADC0~4)는 독립 SAR 컨버터** — 자체 S/H·시퀀서·결과레지스터·ADCINT를 가진다. 인스턴스끼리는 완전히 독립적인 하드웨어.
- **한 인스턴스 안의 여러 SOC = 직렬 변환.** 인스턴스마다 변환기(SAR)·S/H가 **하나뿐**이라, 같은 인스턴스에 채널(SOC)을 여러 개 걸면 **트리거가 같아도 순차 변환**된다. → 채널 간 **샘플 스큐**(뒤 SOC는 앞 SOC 변환시간만큼 늦게 샘플) 발생, 그리고 **모든 SOC 변환시간 합이 트리거 주기 안에 들어와야** 한다. 인터럽트는 **마지막 SOC의 EOC**에 걸어 **한 ISR에서 인스턴스 결과를 coherent하게 read**한다.
- **서로 다른 인스턴스 = 병렬 변환.** 독립 하드웨어라 같은 트리거(예 RTI1)에 걸면 **물리적으로 동시 샘플**된다. 스큐 없음.

## 권장 결정 규칙 (안정성 아님 → 신호 특성 기준)

| 신호 성격 | 권장 배치 | 이유 |
|-----------|-----------|------|
| **동시성 중요 / 빠른 상관 신호** — 전류, V·I 쌍 등 제어루프에 들어가는 것 | **서로 다른 인스턴스 + 공통 트리거** | 동시 샘플 → 위상·전력 계산 정확. 멀티 ADC의 존재 이유 |
| **느린 / 무상관 신호** — 온도, 느린 전압 | **한 인스턴스에 SOC로 몰기, 마지막 EOC에 인터럽트 1개** | 스큐 무의미, 펌웨어 단순(ISR/int_xbar/ready-flag 절약) |

## 펌웨어 복잡도 축

인스턴스를 하나 늘릴 때마다 **+ISR 1개, +int_xbar output 1개, +ready 플래그 1개**가 든다(8kw ADC0 추가가 정확히 이 비용). 그래서 **"동시성이 필요 없는 신호는 한 인스턴스에 먼저 채우는 것"**이 펌웨어를 단순하게 유지하는 타당한 **tiebreaker**다 — 단 **복잡도 절감이지 안정성은 아니다.**

## 유일한 실제 불안정 (overflow)

한 인스턴스에 SOC를 **과적재**해 **변환시간 합 > 트리거 주기**가 되면 → 다음 트리거 전에 시퀀스를 못 끝내 결과가 **밀리거나 덮어씌워진다(overflow).** 이게 유일한 실제 instability이고, **멀티 인스턴스로 분산하면 오히려 해소**된다.

## 8kw 현황 (예시)

현재 [[adc_pinmap]] 배치 — **6채널 완성, 물리 인스턴스 5개**(commit c512e3b 기준):

- **ADC1 = Temp_Module2(SOC0) + GA_Iin_SEN(SOC1)** — 동일 인스턴스 2 SOC, **1 ISR(IRQ146)**. 직렬 변환·스큐 발생.
- **ADC0 = I_COIL_SEN**, **ADC4 = I_LCC_SEN** — 각각 별도 인스턴스(전류 = 상관 신호), SOC0 단독.
- **ADC2 = Temp_Module1**, **ADC3 = GA_Vin** — 각각 별도 인스턴스, SOC0 단독(신규 2채널).
- ⚠️ ADC1이 입력 전류(GA_Iin_SEN, 상관 신호)를 온도(저속)와 동거시켜 직렬 변환됨. **현 보드는 PCB 라우팅 고정이라 그대로 운용**하되, 다음 보드 설계 시 입력 전류는 입력 전압(GA_Vin)과 동시 샘플 가능하도록 별도 인스턴스 배정 검토.

## 한 줄 요약 (다음 보드 설계 시)

상관·고속 신호는 인스턴스를 나눠 **병렬·동시샘플**, 무상관·저속 신호는 한 인스턴스에 모아 **ISR 절약**. "인스턴스 채우는 순서"는 안정성이 아니라 **동시성 요구 + 인스턴스당 변환시간 예산**으로 결정. ★ 단, 실보드에선 **PCB 라우팅이 신호→인스턴스/채널을 거의 고정**하므로, **설계 단계에서** 위 기준으로 핀을 배정해두는 게 핵심.

## 사실 / 가설 / 모름 가름

- **사실 (아키텍처)**: SAR 직렬(한 인스턴스 다중 SOC)/병렬(인스턴스 간)·인스턴스 독립(자체 S/H·시퀀서·결과레지스터·ADCINT). 인스턴스 추가 = +ISR/+int_xbar/+ready-flag.
- **△ 미검증(수치)**: 인스턴스당 **변환시간 예산의 구체 수치 미산정** — overflow 한계를 정량적으로 못 박지 않음.
- **△ 미검증(실측)**: **다중 인스턴스 동시 RTI 트리거의 정밀 동시성 미실측** — "동시 샘플"은 아키텍처 근거이며 스큐 상한을 스코프로 잰 적은 없음.
- 레지스터 수치 출처가 필요하면 [[am263p_trm]] ADC 챕터 demand-ingest.
