# adc_noise_fft_probe — ADC 노이즈 스코프 FFT 진단 + N vs 트리거 위상 결정

> **목적**: 리피터 버스트(N=16, branch `feature/adc-repeater-burst`)의 **최종 N을 정할지, ADC 트리거 위상을 조절할지**를 스코프 FFT 한 번으로 가른다. 로드맵 [[adc]] §A5 "남은 절차 1". 정본 필터 구현 [[am263p_adc_repeater_burst]].
> 회로도 신호체인 출처 [[board_schematic_v1_0e00]], 핀맵 [[adc_pinmap]], 장비 [[instruments]] (MSOX3104T).

---

## 1. 결정 로직 — 노이즈 종류 → 레버 (사용자 모델 + 정합 근거)

| 노이즈 종류 | FFT 모양 | 레버 | 원리 |
|------------|---------|------|------|
| **백색/랜덤** (대역 고루 분포) | 평평한 플로어, 지배 피크 없음 | **N ↑** (오버샘플 평균) | 비상관 노이즈는 √N로 감소 |
| **스위칭 상관** (85 kHz + 고조파 피크) | 85/170/255 kHz…에 뾰족한 피크 | **트리거 위상 조절** (스위칭 순간 회피) | 스위칭 dv/dt·di/dt 과도가 샘플 순간에 주입 |

### ★ 왜 둘이 갈리는가 — 결정적 근거 (사용자 모델 보강)

ADC 트리거는 **EPWM0_SOCA에 위상 고정**(85 kHz 스위칭과 동기). 따라서 버스트 N샘플 전부가 스위칭에 대해 **같은 위상**에서 찍힌다.

- **스위칭 상관 노이즈는 매 주기 동일(coherent)** → N개를 평균해도 그대로 남는다. **√N 평균은 비상관(랜덤) 노이즈에만 듣는다.** ⇒ **스위칭 상관이면 N을 올려도 헛수고**, 샘플 순간을 과도에서 빼내는 **위상**이 유일한 레버.
- 반대로 **백색이면 위상 조절은 무의미**(어느 위상이든 노이즈 동일) → **N만 듣는다**.

→ 사용자의 2분법은 정확. 위 "coherent sampling" 논리가 그 2분법이 **왜** 성립하는지의 핵심.

### 회로도가 더하는 사실 ([[board_schematic_v1_0e00]])

- MCU 핀 입력 RC(100 Ω + ~1.33 nF) **fc ≈ 1.2 MHz** → **85 kHz는 HW 필터가 안 막는다**. 스위칭 상관이면 HW에 기댈 수 없음(보드 캡 증설은 HW 리워크 = 3차 레버).
- 과전류·과전압 보호는 **HW 비교기**(TLV3231 + SN74HCS21 인터록)가 담당 → N을 키워 ADC가 느려져도 **보호 응답은 안 늦는다**(repeater 미채택 논거와 일치, [[am263p_adc_ppb_averaging]]).

## 2. 어디를 찍나 — 프로브 포인트 (회로도 근거)

전부 **DGND 기준 0~3.3 V 저전압 노드** → 프로빙 안전. **GND 클립은 DGND**, PRI_GND/HV 금지([[board_schematic_v1_0e00]] §접지 도메인).

| 채널 | 신호 | 프로브 위치 (LP J3 핀) | 성격 | FFT 적합성 |
|------|------|----------------------|------|-----------|
| **CH1** | **GA_Vin** | **J3.26** (ADC3 AIN0) | DC 버스전압 미러 (U16 절연증폭기 출력) | ★ **최적** — AC 성분 = 순수 노이즈 |
| **CH2** | **GA_Iin_SEN** | **J3.29** (ADC1 AIN1) | 입력 DC 전류 (U1 Hall) | ★ **최적** — DC-nominal |
| (참고) | I_COIL_SEN | J3.28 (ADC0 AIN1) | **정류·평활된 DC 포락선** (CT 1차측이 85 kHz 공진전류) | 순수 노이즈 판별엔 GA_Vin/GA_Iin이 더 깨끗(정류 리플 상재). 단 **FOD 프로브로는 이 노드가 정본** → [[fod_i_coil_observation]] |
| (선택) | Temp_Module1/2 | J3.25 / J3.24 | 저속 NTC | △ 가능하나 제어 비핵심 |

- **GA_Vin·GA_Iin가 1순위** — 정상상태에서 평탄해야 할 DC량이라, FFT에 뜨는 모든 AC가 노이즈로 해석됨.
- **I_COIL_SEN(J3.28)은 정류·평활 DC 포락선** — CT 1차측이 85 kHz 공진전류지만 CR3~6 정류 + C55 RC 평활을 거쳐 ADC 핀에서는 DC. 정류 리플(85 kHz 배수)이 상재해 **순수 노이즈 플로어 판별엔 GA_Vin/GA_Iin이 더 깨끗**하다. 단 **FOD 관찰의 정본 프로브는 이 노드**(이물 → DC 레벨 시프트) → [[fod_i_coil_observation]].
- 핀은 LP J3에서 접근(가장 쉬움). 같은 노드를 eta 보드 J9 측에서 찍어도 됨.

### 타이밍 마커 (위상 결정에 결정적)

| 채널 | 신호 | 위치 | 용도 |
|------|------|------|------|
| **CH3 / D0** | HS1 게이트 = EPWM2_A | **J4.39** ([[pwm_pinmap]]) | **스코프 트리거 소스**(rising) + 스위칭 과도 기준시각 |
| **CH4 / D1** | 디버그 GPIO95 | **J4.31** ([[am263p_lp_debug_gpio]]) | **OSINT ISR(ADC 샘플 순간)에서 토글** → 샘플 시각 마커 |

★ **CH4 = 가장 직접적인 위상 도구**: OSINT ISR(혹은 버스트 시작)에서 디버그 GPIO를 토글하면, 아날로그 노이즈 파형 위에 **"ADC가 여기서 샘플함"** 마커가 겹쳐 보인다 → 샘플이 스위칭 스파이크에 올라타는지 눈으로 확인. MSOX3104T는 MSO(디지털 16채널)라 CH3/CH4를 디지털 핀에 넣어 아날로그 채널을 아껴도 됨.

> ✅ **이 마커는 이미 코드에 구현됨 (`6993a40`, 2026-06-30)**: EOC ISR 진입 시 GPIO95 HIGH / 종료 시 LOW (`eta_bsp_adc.c`). 어느 인스턴스를 마킹할지는 `ETA_BSP_ADC_DBG_MARK_IDX` 한 줄 — **현재 `0U`=I_COIL_SEN(FOD 1차 관찰용 → [[fod_i_coil_observation]], 브랜치 `feature/adc-noise-fft`)**. 여기 노이즈 진단(GA_Vin)을 하려면 **`3U`로 변경**(주석에 GA_Vin=3U·GA_Iin=1U 명시). ⚠️ **잠복 충돌**: `eta_hal_gpio`의 DBG_LOOP도 GPIO95를 써서 후속 정리 대상(`6993a40` 커밋 메시지). 마킹은 **EOC**(버스트 끝) 토글이라, EPWM0_SOCA(버스트 시작) → CH4 엣지 간격이 곧 버스트 변환 구간.

## 3. MSOX3104T 설정 ([[instruments]])

- **프로브**: 10:1 패시브. **반드시 스프링팁 짧은 GND** (긴 악어클립 GND 금지 — 85 kHz 방사 dv/dt를 루프로 주워 FFT에 가짜 피크를 만듦. 인버터 옆 측정의 1순위 실수).
- **수직**: 노이즈가 작으면 **AC 커플링 + V/div 확대**. (MSOX3104T는 8-bit이라 노이즈 플로어 미세 디테일은 RTM3004(10-bit)가 유리 — [[instruments]] 권고. 단 **백색 vs 85 kHz 피크 판별은 8-bit로 충분**, FFT 평균으로 보강.)
- **FFT(Math)**: Hanning 윈도우, 스팬 **0~500 kHz**(또는 1 MHz) — 85 kHz 기본파 + 고조파(170/255/340 kHz…) 보이게. **RMS 평균** 켜서 안정화.
- **트리거**: CH3(HS1 게이트) rising edge.
- **두 화면 다 본다**:
  1. **시간축**: 게이트 엣지로 트리거 → 스위칭 과도 버스트가 11.76 µs 주기 중 **어디**에 있는지 + CH4 샘플 마커가 그 버스트에 올라타는지 → **조용한 창** 위치 파악.
  2. **FFT**: CH1/CH2 노이즈 스펙트럼 → 백색 플로어인지 85 kHz 고조파 피크인지.

### 전제

- **전력단이 실제 스위칭 중**이어야 노이즈가 존재(PWM 활성, 가능하면 대표 전압/부하로 HV 버스 인가). 무동작 idle에서 찍으면 진단 무의미.

## 4. 결정 트리

```
FFT 결과?
├─ 평평한 플로어, 85 kHz 고조파 피크 미미
│   → 백색/랜덤 → N ↑ (ETA_ADC_OVERSAMPLE_LOG2 상향, eta_bsp_adc.h:31)
│     · √N 평균 유효. 단 버스트 N×285 ns ≤ 변환예산(~41, ADC1은 2 SOC 합)
│
├─ 85 kHz + 고조파에 뾰족한 피크 (플로어 위로 우뚝)
│   → 스위칭 상관 → 트리거 위상 조절 (N 상향은 헛수고: coherent)
│     · 레버: EPWM0 SOC 트리거 시점(현재 TBCTR=ZERO)을 옮겨 버스트를
│       스위칭 엣지 사이 "조용한 창"에 배치. 리피터 repPhase/repSyncin.
│     · ★ 주의: N=16 버스트 = 16×285 ns ≈ 4.56 µs ≈ 주기(11.76 µs)의 39%.
│       버스트가 넓어 조용한 창에 안 들어가면 → N도 낮춰 버스트를 좁힌다
│       (즉 위상 + 저N 동시). 순수 either/or 아님.
│
└─ 둘 다 (피크 + 높은 플로어)
    → 위상으로 스위칭 피크부터 빼고, 남은 백색 플로어에 N 적용 (병행)
```
