# fod_i_coil_observation — FOD I_COIL_SEN 1차 관찰

> **목적**: FOD(Foreign Object Detection — 송/수신 코일 사이 이물 감지) 구현의 **1차 단계**. **I_COIL_SEN** 값이 *정상* vs *이물 존재*에서 어떻게 변하는지 관찰하고, 그 값을 **저노이즈로 안정적으로** 잡을 수 있는지 판별한다.
> 브랜치 **`feature/adc-noise-fft`**. 노이즈 판별 로직(백색 vs 스위칭)은 [[adc_noise_fft_probe]]와 공유, 신호체인 출처 [[board_schematic_v1_0e00]], 핀맵 [[adc_pinmap]], 장비 [[instruments]](MSOX3104T), 디버그 GPIO [[am263p_lp_debug_gpio]]. 로드맵 [[adc]] §A5.
> **2026-07-01 갱신**: 외부 스코프(트랙 A) CSV 실측 2회 모두 물리적으로 불가능한 진폭이 재현되어 **보류**, **온타깃 raw ADC 캡처로 주 트랙 전환**(§3.5·§4, 아직 미구현). 현황 [[status]].

## 1. 신호체인 — 왜 이 노드가 FOD 프로브로 적합한가

```
TX 코일전류 → CT(T1 PA6322, 7 mV/A) → CR3~6(CUS10S30) 쇼트키 정류·클램프
            → R30~32 + C55 2.2 µF RC 평활 → J3.28 (ADC0 AIN1)
```

- **★ ADC가 보는 노드는 85 kHz AC가 아니라 정류·평활된 느린 DC 포락선이다.** (RMS `/√2` 환산은 GUI가 정현파 가정으로 하고, 펌웨어는 raw→mV 선형만.)
  → 종전 [[adc_noise_fft_probe]] §2의 "I_COIL_SEN = 85 kHz 신호 자체라 부적합" 서술은 **CT 1차측(센싱 대상)** 기준이었고, **J3.28 노드(ADC 입력)는 DC 포락선**이라는 점을 정정.
- **FOD 가설**: 이물이 코일 결합·부하를 바꾸면 코일전류가 변하고 → 이 **DC 레벨 시프트**로 나타난다. 그래서 이 노드가 이물 판별 프로브로 적합.
- ADC0은 EPWM0 SOCA(스위칭 주기마다 TBCTR=ZERO) 트리거 → 리피터 버스트 N=16 블록평균 → **주기당 값 1개(≈85k samples/s)**. 정본 [[am263p_adc_repeater_burst]].
- **⚠️ OC 제약**: 같은 센서 노드를 `I_COIL_OC` 과전류 비교기(TLV3231)가 탭한다. 이물 실험은 **반드시 OC 임계 이하**로 유지.

### ★ 왜 I_COIL_SEN *단독* 으로 백색 vs 스위칭 상관을 판별해도 되나

[[adc_noise_fft_probe]]는 순수 노이즈 판별엔 GA_Vin/GA_Iin이 더 깨끗하다고 적지만, **FOD 목적의 백색/상관 판별은 I_COIL_SEN 단독으로 타당하다** — 오히려 더 직접적이다:

1. **이기려는 노이즈가 곧 I_COIL_SEN의 노이즈다.** 검출성 `Δ/σ`의 σ는 I_COIL_SEN 자신의 노이즈 → 정제 대상 위에서 직접 봐야 결정이 바로 행동이 된다. GA_Vin의 노이즈 성격으로 I_COIL_SEN 정제를 정하는 건 간접적.
2. **정상·정상상태에서 이 노드는 공칭 DC다.** 이물 없고 부하 고정이면 코일전류 일정 → 정류·평활 DC 포락선도 일정. FFT 대역(수 kHz~500 kHz)은 C55 평활 코너보다 훨씬 위라 **그 대역의 모든 AC = 노이즈**. GA_Vin을 좋은 프로브로 만든 성질(공칭 DC량)과 동일.
3. **백색/상관 이분법이 노드와 무관하게 치료법으로 직결.** 평탄 플로어→비상관→N↑ / 85·170 kHz 피크→EPWM 고정 샘플링과 coherent→위상. §3 결정과 동일.
4. **유일한 단서(정류 리플)는 결정을 못 뒤집는다.** I_COIL_SEN엔 정류 리플(85 kHz 배수)이 남아, 결합 노이즈가 아니어도 85 kHz 계열 피크가 뜰 수 있다. **그러나 리플이든 스위칭 결합 노이즈든 둘 다** 스위칭에 coherent + 샘플링에 위상고정 → √N 무효 → **같은 레버(트리거 위상/조용한 창)** 를 요구한다. 피크의 정체를 구분 못 해도 **취할 행동은 동일** → 결정 트리는 이 혼동에 강건. 백색-플로어 vs 피크가 판별할 유일한 것은 "평균으로 주는가 아닌가"이고 I_COIL_SEN은 그걸 충실히 답한다.

> **경계**: "GA_Vin이 더 깨끗"은 **절대 노이즈 플로어 dB를 정밀 측정하는 순수 노이즈 연구**(정류 리플 없음 + 스코프 8-bit 한계 회피)에서의 얘기 — 이 결정의 타당성과는 별개다. 순수 플로어 수치가 궁금하면 GA_Vin/GA_Iin, **FOD go/no-go·평균 vs 위상 결정**은 I_COIL_SEN.
>
> **실무 보너스**: 디버그 마커(GPIO95/CH4)가 이미 ADC0=I_COIL_SEN(`idx 0U`)에 걸려 있어 샘플 순간(CH4)·스위칭 엣지(CH3)·노이즈(CH1)를 **같은 채널 위에서 coherent**하게 본다 → 샘플이 스파이크에 올라타는지 직접 확인.

## 2. 측정 셋업 — 2 트랙 동시 캡처

전제: **전력단 실제 스위칭 중**(PWM 활성 + HV 버스 인가). idle이면 신호/노이즈 없어 측정 무의미. GND 클립은 **DGND에만**(PRI_GND/HV 직접 프로빙 금지 — 절연 거친 0~3.3 V 저전압 노드).

### 프로브 맵 (MSOX3104T, 10:1 패시브 + 짧은 스프링팁 GND)

| CH | 신호 | 위치 | 용도 |
|----|------|------|------|
| **CH1** | **I_COIL_SEN** | **J3.28** | FOD 1차 신호. DC=레벨, AC=노이즈 (트랙 A) |
| CH3 | EPWM2_A (HS1) | J4.39 | 스코프 트리거(rising) + 스위칭 기준 시각 |
| CH4 | GPIO95 | J4.31 | ADC0 버스트-끝 마커 (주기당 펄스 1개, 상승엣지=샘플링 끝) |

- **디버그 마커**: `src/bsp/eta_bsp_adc.c`의 `#define ETA_BSP_ADC_DBG_MARK_IDX 0U` → EOC ISR이 **ADC0(I_COIL_SEN)** 버스트 완료 시 GPIO95 토글(진입 HIGH/종료 LOW). 다른 채널은 값만 바꿔 재빌드(`3U`=GA_Vin, `1U`=GA_Iin). **측정용 디버그 스캐폴딩**(측정 후 제거 대상). ⚠️ `eta_hal_gpio` DBG_LOOP도 GPIO95 사용 → 잠복 충돌, 후속 정리.

### 트랙 A — 스코프 (풀레이트 아날로그, J3.28의 풀밴드 진실)

같은 CH1 프로브를 **커플링만 토글**해 본다:
1. **DC 커플링** — 절대 레벨. 정상 baseline 기록 → 이물 투입 시 레벨 시프트(Δ) 직접 관측 = **FOD 신호 본체**.
2. **AC 커플링 + V/div 확대** — DC 빼고 그 위 리플·노이즈 확대.
3. **FFT(Math)** — Hanning, 0~500 kHz, RMS 평균, 트리거 CH3 rising. **평탄 플로어=백색** / **85·170·255 kHz 피크=스위칭 상관**.
4. **시간축** — CH3 스위칭 엣지 ↔ CH4 상승엣지 간격 = 샘플링 구간(이론 N=16 → 4.56 µs). 샘플링 창이 스위칭 과도·리플 위에 올라타는지 확인.

#### CSV export 설정 (오프라인 FFT용) — 방법론 일반화는 [[instruments]] 참조

라이브 Math FFT 대신 CSV를 뽑아 Python으로 FFT/σ를 계산하려면: Save/Recall → Save → 포맷 CSV, Length로 저장 점수(N) 지정.

**핵심 규칙(MSOX3104T 일반, [[instruments]]에 방법론으로 등록)**: 고정 점수 N에서 **유효 Fs = N ÷ 창길이(T)** 를 결정하는 건 **타임베이스(time/div, 창 T = 10 div)** 다 — 점수(N)만으론 Fs가 안 정해진다. 화면의 `Sa/s` 표시로 항상 확인. 이 프로젝트는 목적별로 창을 **두 번** 다르게 떴다:

| 프로파일 | 목적 | 타임베이스 (창 T) | 필요 Fs (Nyquist) | 얻는 것 |
|---|---|---|---|---|
| **A. 저속 포락선** | DC 레벨·드리프트·저주파 노이즈 σ | 수십 ms/div (창 수백 ms~1 s) | 수 kS/s | 0~1 kHz σ·베이스라인 (85 k **불가**) |
| **B. 고속 스위칭** | 리플 형상 + 85/170/255 kHz FFT | 20~50 µs/div (창 200~500 µs) | ≥ 2.5 MS/s (Nyquist ≥ 1.25 MHz) | 스위칭 하모닉·리플 |

- **앨리어싱 방지**: Length를 줄여 저장하면 Fs_saved = N/T가 떨어져 HF가 접힌다. 저장 후 반드시 **N/T ≥ 목표 Nyquist의 2배** 재확인. (이 세션의 초기 `dc.csv`/`ac.csv`는 창 1 s·2000점 → Fs 2 kS/s라 85 k가 앨리어싱됨 — **Profile A 용도로만** 유효.)
- 트리거는 CH3 EPWM rising 유지 → 위상고정 코히런트 캡처라 스위칭 피크가 날카롭다.

### 트랙 B — 디지털 (PC 모니터, 10 Hz)

- `python3 tools/gui/launch.py` → Port 선택 → Connect. **I_COIL_SEN = ch4** 라이브(V·A).
- **Start Log** → `wpt_log_<ts>.csv`(헤더 `timestamp, seq, <채널>_raw`, raw counts 10 Hz) → **Stop Log** → `scratch/`로 이동.
- 10 Hz는 UART 대역폭 한계 — **정상상태 레벨/델타엔 충분, 빠른 과도엔 부족**(풀레이트 디지털은 §4로 미룸).

## 3. 절차 · 판정

> **채택 실행순서 = 안정화 선행 (2026-07-01)**: 아래 "정상→이물→(필요시)안정화" 절차 대신, **① 정상에서 노이즈 측정 → ② N/위상 튜닝으로 σ 선(先)감소 → ③ 안정화된 σ 위에서 이물 Δ 관찰** 순서로 진행한다. σ를 먼저 낮춰야 `Δ/σ`가 유리하고, 안정화 튜닝을 이물 변수 없이 깨끗하게 할 수 있다. 아래 1·2는 ③에 해당(정상 캡처는 ①과 공유).

각 조건에서 트랙 A(스코프)와 트랙 B(CSV)를 **동시** 캡처:

1. **정상(이물 없음)**: DC 레벨·AC 노이즈·FFT·시간축 캡처 + CSV 로깅 → I_COIL_SEN_raw **평균·표준편차 σ**(노이즈 플로어, counts).
2. **이물 투입**(OC 임계 이하): 같은 항목 재캡처 → 레벨 시프트 **Δ counts**·노이즈 변화 기록.
3. **검출성 = `Δ(이물) / σ(정상 노이즈)`**:
   - 충분히 크면 → 현 센싱으로 FOD 가능, 임계 설계로.
   - 마진 부족 → 센싱 안정화 필요(아래).

### 센싱 안정화 결정 (FFT 노이즈 성격으로 가름 — [[adc_noise_fft_probe]] §4 공유)

- **백색(평탄 플로어)** → 오버샘플 N 상향(`eta_bsp_adc.h:31` `ETA_ADC_OVERSAMPLE_LOG2`). 단 버스트 N×285 ns ≤ 변환예산(~41, ADC1은 2 SOC 합).
- **스위칭 상관(피크)** → 트리거 위상 조절(N↑은 헛수고 — 위상고정→coherent→√N 무효). EPWM0 SOCA 시점(현재 TBCTR=ZERO)을 조용한 창으로, 또는 리피터 repPhase(`eta_bsp_adc.c`).

## 3.5 결정 — 트랙 A(외부 스코프) 보류, 온타깃 raw 캡처로 전환 (2026-07-01)

Keysight CSV export로 **2회 실측**(`dc.csv`/`ac.csv`, §2 Profile A/B 절차 적용)했으나 **두 시도 모두 진폭이 물리적으로 불가능한 범위**(pk-pk 20~36 V)로 나왔다 — J3.28은 절연 거친 **0~3.3 V 저전압 노드**라 이 값 자체가 성립할 수 없다.

| 시도 | 설정 | 결과 |
|------|------|------|
| 1차 | Fs 40 MS/s, 창 50 µs | Fs는 충분했으나 창이 짧아 85 kHz **4.25 사이클뿐**(FFT 분해능 부족). 진폭도 비정상 |
| 2차 | 창 5 ms | Fs 0.4 MS/s로 떨어져 170/255 kHz 관측 불가(앨리어싱). 진폭 문제는 **재현** |

**판정**: 두 시도 모두 진폭 이상이 재현됐다는 사실로부터 **프로브/V-div/감쇠비 설정 오류가 아니라 하드웨어 측정 한계**로 판단한다. ⚠️ **원인은 미규명 — 빈자리로 남긴다**(프로브 임피던스 부정합·GND 루프·스코프 채널 결함·감쇠 설정 오기 등 후보를 확인 없이 단정하지 않음).

**결정**: 외부 스코프로 J3.28 절대 레벨을 신뢰성 있게 잡기 어려워, **트랙 A(외부 스코프)는 보류**하고 **MCU 내부에서 직접 raw ADC 값을 뽑는 방식**(§4 온타깃 버스트 캡처 + UART 덤프)을 1차 관찰의 **주 트랙**으로 전환한다.

## 4. 다음 트랙 — 온타깃 raw ADC 캡처 (2026-07-01 승격, ⚠️ 미구현)

ADC는 이미 ~85k samples/s를 만들지만 텔레메트리는 10 Hz만 떠간다. 원래는 "스코프(아날로그 풀레이트) + 10 Hz(정상상태 레벨)로 충분하다"고 보고 이 항목을 다음 작업으로 미뤘으나, §3.5에서 외부 스코프 진폭이 물리적으로 불가능한 값으로 재현되어(원인 미규명, 측정 한계로 판단) **트랙 A를 보류** — 이 항목이 **1차 관찰의 주 트랙으로 승격**됐다.

- **필요한 것**: 온타깃 버스트 캡처(예: 연속 2048 샘플 링버퍼) + **1회 UART 일괄 덤프**(연속 UART 스트리밍은 대역폭상 불가). MCU 내부에서 직접 raw ADC 값을 뽑아 절대 레벨·노이즈 σ를 확보.
- **⚠️ 빈자리 — 아직 미구현**: 캡처 트리거 방식·버퍼 크기·덤프 프로토콜(패킷 포맷)·PC측 수신·FFT 파이프라인 전부 미정. 다음 세션 착수 대상.
- 런타임 FOD 로직을 타깃에서 최종 구동하면 칩 내부에서 풀레이트를 공짜로 쓰므로, 이 PC 풀레이트 캡처는 **개발·검증용**일 뿐 — 최종 FOD 판정 경로가 아님.

- 온타깃 raw 캡처(§4) 구현 후: 캡처·덤프 프로토콜은 [[am263p_adc_repeater_burst]] 인접 위치에, FFT/σ 결과·검출성(Δ/σ)은 이 페이지에 환원.
- 위상을 택했다면 EPWM0 SOC 트리거 시점 변경 구현은 [[am263p_adc_repeater_burst]]·[[am263p_adc_rti_trigger]] §5에 환원.
