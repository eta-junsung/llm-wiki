## 1. 이 작업이 뭔가

**목표**: eta 자체 설계 **8kW EV 무선전력전송(WPT) 송신 보드(Ver1.0E00)** 펌웨어 개발.
**플랫폼**: LP-AM263P (TI MCU 보드)가 J3 커넥터로 eta 보드와 연결.
**이번 첫 작업 호**: **ADC 계측 브링업** — 보드 6채널(온도·전압·전류) 아날로그 신호를 MCU가 읽어 수치화.

> **한 줄 비유**: 송신 보드의 온도·전압·전류 센서들을 MCU의 "계기판"에 연결하는 작업. 지금은 계기판이 숫자(전압)를 정확히 읽는 데까지 완성.

## 2. 작업 호 — A0~A4

```
[A0] ADC SysConfig 설정                  ✓
[A1] 단채널 실보드 검증                   ✓
[A1.5] UART 출력 주기화 + 코드 리팩토링   ✓
[A2] 6채널 전부 읽기 (실보드 검증)        ✓  ← 이번주 완료
        ▼
[A3] 신호별 물리량(°C/V/A) 변환          ?  ← 센서 스펙 대기(블로커)
[A4] 멀티미터 기준 교차검증              ✗
```

상태: `✓` 완료+검증 / `?` 추가 정보 필요 / `✗` 미완

추가로, ADC와 별개로 **UART5 차동(RS-485) 송신**은 아직 미동작(§4-2).

### 신규 작업 호 — PWM 전력제어 (P0~P4)

ADC 계측이 끝나, 다음은 **PWM으로 송신 인버터를 구동**하는 전력제어 트랙. **핀맵 확정 → P1 착수 가능.**

```
[P0] 요구사항·핀맵 확정   △ ← 4핀 채널 확정·게이트극성 active-high 실증 / 주파수값·보호신호·극성 회로도만 미정
[P1] 기본 PWM 출력        ✓ ← 4채널(HS1/LS1/HS2/LS2) 실보드 검증, shoot-through 0 (6e6b342)
[P2] dead-time 튜닝       △ ← 두 레그 ETA_DEADTIME_NS 단일소스 통일·150/300ns 스윕 4ch 실측 통과 (8046744) / 최종값 스펙 대기
[P3] 보호 (과전류/과전압 시 PWM 즉시 차단)  ✗
[P4] ADC 피드백 제어 + ADC 트리거를 RTI→PWM 전환  ✗
```

**확정된 스펙(사용자 제공 + UG 교차확인)** — 핀맵 정본 [[pwm_pinmap]]:
- **풀브리지 4채널, EPWM 3모듈**: 레그1=EPWM2(HS1 J4.39 / LS1 J4.40), 레그2=**EPWM4(HS2 J6.52) + EPWM7(LS2 J6.51)**. UART5(EPWM15) 충돌 없음.
- 스위칭 주파수 **고정형**(값 추후), **dead-time만 가변** — 리얼타임 아니라 값 바꿔 빌드 반복, **시작 150ns**.
- ADC 트리거는 **지금은 RTI 유지**, PWM 완료 후 **PWM을 트리거 소스로 전환 예정**.

> ⚠️ **레그2가 두 EPWM 모듈에 걸쳐 있음**(보드 라우팅) → 레그2 dead-time을 모듈 내 기능이 아니라 **두 모듈 동기+위상**으로 만들어야 함(레그1보다 까다로움). 의도된 현 설계이며, **향후 회로도 수정 기회 시 한 모듈로 묶도록 요청 예정**. ADC 잔여(A3) 센서 스펙·PWM 주파수를 함께 확보하면 두 트랙이 같이 풀림.

## 3. 이번주 진행 & 만난 문제

### 진행

- 단채널 → **6채널 ADC 계측 완성**, 실보드에서 raw→전압(mV) 변환 경로 검증 완료.
- 6채널을 물리 ADC 5개에 배치(빠른/상관 신호는 동시 샘플 가능하도록 분산).
- ADC 처리 코드를 **테이블 주도 구조로 리팩토링**(중복 통합, 약 100줄 감소) — 향후 채널 추가가 "표 1행"으로 끝남.

### 만난 문제 & 해결 (보고용)

| 문제 | 원인 | 해결 |
|------|------|------|
| 타이머로 트리거를 걸어도 ADC 변환이 한 번도 안 일어남 | SysConfig에서 타이머 인터럽트(`enableIntr0`) 미설정 → 트리거 이벤트가 ADC까지 전달 안 됨 | 설정 한 줄 활성화로 즉시 해결 (SW 강제변환으로 경로 생존 먼저 확인해 원인 좁힘) |
| 채널 추가 시 ADC가 엉뚱한 핀 값을 읽음 (인터럽트·변환은 정상인데 인가전압 미추종) | 핀 자동배정(soft)이 **새 채널 추가 때마다 기존 배정을 재셔플** | 물리 인스턴스 + **AIN 핀까지 전부 수동 고정(hard 배정)** 으로 차단 |

> 두 번째 함정은 BP-CC3351 포팅에서 겪은 동일 유형(자동 핀배정의 재셔플)으로, 그 경험이 8kw에 바로 적용됐음.

## 4. 지금 어디 / 막힌 곳

### 4-1. 완료

- **ADC 계측 6채널 완성·검증** (현재 출력은 콘솔 UART로 6채널 표시, raw→mV까지).

### 4-2. 막힌(대기) 항목 2가지

1. **신호별 물리량 변환(A3) — 센서 스펙 미입수로 블로커**: 지금은 raw→전압(mV)까지만. 온도(V/°C)·전압(분압비)·전류(mV/A·오프셋) 변환식을 쓰려면 **센서 사양 입수 필요**.
2. **UART5 차동(RS-485) 송신 미동작**: 콘솔 출력은 되나, 차동라인 송신 코드가 비활성 + RS-485 트랜시버(THVD1400) 제어 미구현. (펌웨어 핀 설정 자체는 정상 확인 — 트랜시버 인에이블 제어가 남음)

## 5. 다음 작업 — 어떻게 진행할 예정인가

**다음 활성 트랙: PWM (전력제어).** 핀맵·스펙 확정으로 **P1(기본 PWM 출력) 착수 가능**.

1. **PWM — P1 완료·P2 dead-time 단일소스 통일(완료)**: EPWM2/4/7 4채널 실보드 출력 검증(100kHz·shoot-through 0). 레그2는 두 모듈 SYNC 상보. dead-time은 **두 레그 모두 `ETA_DEADTIME_NS` #define 하나**로 조절(레그1 dead-band RED/FED·레그2 CMPB 오프셋), 150/300ns 스윕 4ch 실측 통과. **다음: P3 보호(trip-zone) → P4 제어루프+ADC 트리거 전환.** 게이트 극성·주파수·보호신호 스펙 확보 선결.
2. **A3 센서 스케일링** — 센서 사양 입수 후 물리량 변환. *(PWM 주파수 확정값과 함께 확보 권장)*
3. **UART5 차동 송신 복구** — 송신 코드 활성화 + RS-485 인에이블(DE) 제어 구현·검증.
4. **A4 실보드 교차검증** — 멀티미터 기준값으로 6채널 출력 오차 정량화.

> 실작업 우선순위: **PWM P1 착수** 를 다음 주 목표로. 센서 스펙·PWM 주파수값을 같이 받아오면 A3 병행 가능.
