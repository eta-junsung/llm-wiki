## 1. 이 작업이 뭔가

**목표**: eta 자체 설계 **8kW EV 무선전력전송(WPT) 송신 보드(Ver1.0E00)** 펌웨어 개발.
**플랫폼**: LP-AM263P (TI MCU 보드)가 J3 커넥터로 eta 보드와 연결.
**이번 첫 작업 호**: **ADC 계측 브링업** — 보드 6채널(온도·전압·전류) 아날로그 신호를 MCU가 읽어 수치화.

> **한 줄 비유**: 송신 보드의 온도·전압·전류 센서들을 MCU의 "계기판"에 연결하는 작업. 지금은 계기판이 숫자(전압)를 정확히 읽는 데까지 완성.

## 2. 작업 호 — A0~A4

```
[A0] ADC SysConfig 설정                  ✓
[A1] 단채널 실보드 검증                   ✓
[A1.5] UART 출력 주기화 + 코드 리팩토링   ✓
[A2] 6채널 전부 읽기 (실보드 검증)        ✓  ← 이번주 완료
        ▼
[A3] 신호별 물리량(°C/V/A) 변환          ?  ← 센서 스펙 대기(블로커)
[A4] 멀티미터 기준 교차검증              ✗
```

상태: `✓` 완료+검증 / `?` 추가 정보 필요 / `✗` 미완

추가로, ADC와 별개로 **UART5 차동(RS-485) 송신**은 아직 미동작(§4-2).

### 신규 작업 호 — PWM 전력제어 (P0~P4, 미착수)

ADC 계측이 끝나, 다음은 **PWM으로 송신 인버터를 구동**하는 전력제어 트랙.

```
[P0] PWM 요구사항·핀맵 확정 (토폴로지·채널·주파수·dead-time·보호)  ✗ ← 스펙 대기(첫 게이트)
[P1] 기본 PWM 출력 (오실로 검증)                                   ✗
[P2] 다채널·위상·dead-time                                         ✗
[P3] 보호 (과전류/과전압 시 PWM 즉시 차단)                          ✗
[P4] ADC 피드백 제어 루프                                          ✗
```

> ADC 잔여(A3)도 센서 스펙 대기라, **PWM 스펙과 센서 스펙을 함께 확보**하면 두 트랙이 같이 풀림.

## 3. 이번주 진행 & 만난 문제

### 진행

- 단채널 → **6채널 ADC 계측 완성**, 실보드에서 raw→전압(mV) 변환 경로 검증 완료.
- 6채널을 물리 ADC 5개에 배치(빠른/상관 신호는 동시 샘플 가능하도록 분산).
- ADC 처리 코드를 **테이블 주도 구조로 리팩토링**(중복 통합, 약 100줄 감소) — 향후 채널 추가가 "표 1행"으로 끝남.

### 만난 문제 & 해결 (보고용)

| 문제 | 원인 | 해결 |
|------|------|------|
| 타이머로 트리거를 걸어도 ADC 변환이 한 번도 안 일어남 | SysConfig에서 타이머 인터럽트(`enableIntr0`) 미설정 → 트리거 이벤트가 ADC까지 전달 안 됨 | 설정 한 줄 활성화로 즉시 해결 (SW 강제변환으로 경로 생존 먼저 확인해 원인 좁힘) |
| 채널 추가 시 ADC가 엉뚱한 핀 값을 읽음 (인터럽트·변환은 정상인데 인가전압 미추종) | 핀 자동배정(soft)이 **새 채널 추가 때마다 기존 배정을 재셔플** | 물리 인스턴스 + **AIN 핀까지 전부 수동 고정(hard 배정)** 으로 차단 |

> 두 번째 함정은 BP-CC3351 포팅에서 겪은 동일 유형(자동 핀배정의 재셔플)으로, 그 경험이 8kw에 바로 적용됐음.

## 4. 지금 어디 / 막힌 곳

### 4-1. 완료

- **ADC 계측 6채널 완성·검증** (현재 출력은 콘솔 UART로 6채널 표시, raw→mV까지).

### 4-2. 막힌(대기) 항목 2가지

1. **신호별 물리량 변환(A3) — 센서 스펙 미입수로 블로커**: 지금은 raw→전압(mV)까지만. 온도(V/°C)·전압(분압비)·전류(mV/A·오프셋) 변환식을 쓰려면 **센서 사양 입수 필요**.
2. **UART5 차동(RS-485) 송신 미동작**: 콘솔 출력은 되나, 차동라인 송신 코드가 비활성 + RS-485 트랜시버(THVD1400) 제어 미구현. (펌웨어 핀 설정 자체는 정상 확인 — 트랜시버 인에이블 제어가 남음)

## 5. 다음 작업 — 어떻게 진행할 예정인가

**다음 활성 트랙: PWM (전력제어).** ADC 잔여는 스펙/HW 대기라 PWM이 다음 진행 트랙.

1. **PWM P0 — 요구사항·핀맵 확정 (착수 첫 게이트)**: 토폴로지·채널 수·스위칭 주파수·dead-time·위상·보호신호를 보드 설계자료에서 입수, EPWM 핀 배정(UART5 점유 핀 회피). 이후 P1 기본출력 → P2 다채널·dead-time → P3 보호 → P4 제어루프.
2. **A3 센서 스케일링** — 센서 사양 입수 후 물리량 변환. *(PWM 스펙과 함께 확보 권장)*
3. **UART5 차동 송신 복구** — 송신 코드 활성화 + RS-485 인에이블(DE) 제어 구현·검증.
4. **A4 실보드 교차검증** — 멀티미터 기준값으로 6채널 출력 오차 정량화.

> 실작업 우선순위: **PWM 착수(P0 스펙 확보)** 를 다음 주 목표로. 센서 스펙도 같이 받아오면 A3까지 병행 가능.
