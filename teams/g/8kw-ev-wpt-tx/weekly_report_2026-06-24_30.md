## 2. ADC 트리거 전환 + HW 평균 필터 (A3.5, PR #6, 실보드 검증)

들어오는 ADC 값의 노이즈(널뛰기)를 줄이기 위해, 트리거 소스를 바꾸고 하드웨어 평균을 도입했다.

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| SOC 트리거 | RTI1 타이머 (1 kSPS) | **EPWM0_SOCA (85.032 kHz)** — PWM 마스터 클럭 |
| 노이즈 처리 | 없음 (raw) | **PPB 누적 HW 평균 N=64** (CPU 부하 0) |

- 6채널 전부 PWM 마스터(EPWM0)에 물려 샘플링이 스위칭 주파수(85 kHz)에 동기.
- 평균은 **PPB(Post-Processing Block) 하드웨어 누적** 방식 — CPU 개입 없이 노이즈가 √64 = 1/8로 감소.
- 평균 횟수 N은 매크로 **한 곳**(`ETA_ADC_OVERSAMPLE_LOG2`)으로 제어(6→64, 5→32 …), 코드 수정 후 재빌드만으로 조정.
- **실보드 HW 노이즈 측정으로 N=64 확정.** 6채널 0/3.3 V 추종·ISR 레이트 실측 OK.

## 3. ADC 필터 방식 전환: 누적 평균 → 리피터 버스트 블록평균 (A5, PR #11, 실보드 + Saleae 검증)

§2의 PPB 누적 평균(N=64)은 노이즈는 잘 줄지만, 트리거당 여러 주기에 걸쳐 값이 나와 **실효 출력 레이트가 1.33 kHz로 떨어지는** 한계가 있었다. 이를 **한 트리거 안에서 16회 연속 변환 → 블록평균**하는 "리피터 버스트" 방식으로 전환했다.

- **효과**: 평균(노이즈 억제)은 유지하면서 **출력 샘플레이트가 다시 85 kHz/인스턴스로 복귀**(N과 무관).
- main 머지 완료(**PR #11**, CI 통과), 실보드에서 6채널 라이브 + 스코프 OSINT 85 kHz 검증.
- 현재 운영값 **N=16** (최종 N은 아래 §4 FFT 측정 후 튜닝).

### Saleae 로직 분석기 실측 (6/30~7/1)

버스트 방식이 실제로 "한 트리거 = 한 번 완결"로 동작하는지 로직 분석기로 실증했다.

| 측정 항목 | 결과 |
|-----------|------|
| 트리거당 ISR 발생 횟수 | **정확히 1회** (12,260 / 12,260 트리거, 누락·중복 0) → 버스트 실증 |
| 유효 샘플레이트 | **85.03 kHz** (누적 방식이었다면 ≈5.3 kHz여야 함) |
| 트리거 → 버스트 종료 | 3.12 µs (주기의 26.5%) |
| ISR 처리 시간 | 0.304 µs |

> 만약 종전 누적(N=64) 방식이었다면 ISR이 트리거마다 나오지 않았을 것 — 트리거당 정확히 1회 ISR이 곧 버스트(블록평균)가 맞다는 직접 증거.

### 변환시간 예산 정밀 산정 (리피터 N 상한)

버스트 N을 얼마까지 키울 수 있는지 산정했다. 1회 변환 cadence ≈ **285 ns**(S+H 80 ns + EOC 205 ns), 85 kHz 주기(11.76 µs)당 **~41 변환** 수용 가능 → **리피터 N 상한 ~41**. 즉 N=16은 예산 안에 충분히 여유가 있어, FFT 결과에 따라 상향 여지가 있다.

## 4. 8kW 보드 회로도 분석 + 노이즈 측정 계획 수립 (6/30)

다음 단계(최종 N / 트리거 위상 결정)를 측정으로 판단하기 위해, 보드 회로도(Ver 1.0E00)를 분석해 ADC 센서 신호체인을 파악했다.

### ADC 센서 신호체인

| 신호 | 센서 | 측정 대상 |
|------|------|-----------|
| GA_Vin | U16 AMC0311 절연 증폭기 | DC 버스 전압 |
| GA_Iin | U1 TMCS1126 Hall 센서 | 입력 DC 전류 |
| I_COIL | T1 PA6322 CT | 85 kHz 공진 전류 |
| Temp ×2 | 모듈 NTC | 모듈 온도 |

- MCU 핀 입력 RC 필터(100 Ω + ~1.33 nF) 차단주파수 fc ≈ 1.2 MHz → **85 kHz 신호는 감쇠되지 않고 통과.**
- HW 보호 인터록(비교기 → GD_EN) 경로도 회로도에서 확인.

### 노이즈 측정 계획 (FFT 프로브)

- **판별 로직**: 노이즈가 백색(랜덤)이면 → 평균 N을 키우면 개선. 스위칭 주파수(85 kHz) 고조파와 상관되면 → ADC 트리거가 EPWM에 위상 고정돼 있어 평균이 무효 → **트리거 위상**이 유일한 레버.
- **프로브 포인트**: GA_Vin(J3.26)·GA_Iin(J3.29) 같은 DC 신호핀이 적합. I_COIL은 85 kHz가 신호 자체라 부적합.
- 장비·설정(MSOX3104T, Hanning·0~500 kHz·RMS 평균)까지 확정.

## 5. FOD(이물질 감지) 1차 관찰 착수 (`6993a40`, 6/30)

무선충전 이물질 감지(Foreign Object Detection) 선행 관찰로, EOC ISR에서 **디버그 GPIO(GPIO95, J4.31)** 를 토글해 I_COIL_SEN 버스트 완료 시각을 스코프로 관측하는 마커를 넣었다. 코일 전류 노이즈/안정성 판별의 첫 단계.

⚠️ `eta_hal_gpio`의 다른 디버그 루프도 GPIO95를 써서 잠복 충돌 — 후속 정리 필요.

## 다음 작업 — FOD I_COIL_SEN 1차 관찰

이번 주 착수한 FOD 관찰(§5)을 정식 절차로 수행한다. **이물(금속편)이 코일 결합·부하를 바꾸면 코일전류가 변하고, 그것이 I_COIL_SEN의 DC 레벨 시프트로 나타난다**는 가설을, 정상 vs 이물 상황을 비교 측정해 검증한다.

- **신호체인 파악**: I_COIL_SEN(J3.28)은 코일전류 CT(T1, 7 mV/A) → 쇼트키 정류 → RC 평활을 거친 **DC 포락선**. 이물에 의한 레벨 변화 관측에 적합.
- **측정 방법 (2트랙 동시)**:
  1. **스코프(풀레이트)** — DC 커플링으로 정상 baseline 대비 이물 투입 시 레벨 시프트(Δ), AC 커플링·FFT로 노이즈 성격(백색 vs 85 kHz 스위칭 상관) 확인.
  2. **PC GUI(10 Hz)** — I_COIL_SEN raw를 CSV 로깅해 정상 평균·표준편차(σ) 산출.
- **판정 지표**: 검출성 = 이물 레벨 시프트 Δ / 정상 노이즈 σ. 마진이 충분하면 임계 설계로, 부족하면 센싱 안정화(노이즈가 백색이면 평균 N 상향 / 스위칭 상관이면 트리거 위상 조절).

### 후속 (FOD 이후)

1. **SW 이동평균 필터(A6)** — HW 버스트 평균 위에 SW ring buffer 이동평균을 얹어 스파이크+노이즈 동시 억제.
2. **ADC A4 실보드 교차검증** — 멀티미터 기준값으로 스케일링 정확도 확인.
3. (병행) **PWM P3 보호(trip-zone)** 착수 검토.
