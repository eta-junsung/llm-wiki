3. 현재 구현 중 — SPI_Comm_St (HDR 0x10 Buffer[0] Bit.5)

필요한 이유: SPI는 Master(STM32)가 클럭을 주도하는 구조라, Slave(nRF)가 죽어도 Master는 계속 전송하며 정상으로 인식한다. STM32(01_RX_control)가 nRF(02_RX_esb)와의 SPI 연결이 살아있는지 스스로 알 수 없기 때문에 heartbeat 비트가 필요하다.

항목내용
위치패킷 HDR 0x10 — Buffer[0] Bit.5
생성 주체02_RX_esb (nRF52)
동작200 ms마다 0 → 1 → 0 교번
감시 주체01_RX_control (STM32)
단절 판정일정 시간 비트 변화 없음 = SPI 단절
상황처리
대기 상태Warning
출력 중Fault + 출력 차단
⚠️ SPI_Comm_St는 nRF ↔ STM32 SPI 구간 헬스체크 (무선 품질과 무관).
BLE_Comm_St(Bit.6)는 무선 링크 상태 — ESB 전환 후 의미 재정의 필요. 두 비트 혼동 주의.

4. ESB 타이밍 측정 (오실로스코프 실측, 2026-05-27)

측정 장비: Keysight MSO-X 3104T  |  프로브 핀: P0.17 (TX 시작), P0.18 (ACK 수신)  |  대상: 03_TX_esb (esb 브랜치)

측정 1 — TX→ACK 왕복 지연: 약 470 μs
측정 2 — TX 전송 주기: 약 920 μs
측정 3 — ACK 수신 주기: 약 940 μs

5. 미결 사항

  • 실보드 장시간 안정성 검증 — TX_FAILED 카운터 동작 확인
  • GPIO 토글 핀(P0.17/P0.18) 제거 여부 결정 (검증 완료 후)
  • 01_RX_control ↔ 02_RX_esb UART 브리지 연동 동작 확인
  • SPI_Comm_St 구현 완료 + 실보드 검증
