> 한 작업(task) 단위 호. 프로젝트 전체 호는 [[roadmap]], 기능별 현황·다음 시작점은 [[status]].

---

## 0. 한 줄 요약

PC에서 **UART로 받은 패킷 데이터를 모니터링** + **`buck` 커맨드를 설정**하는 간단한 GUI. 펌웨어 변경 없이 기존 UART 인터페이스 위에 얹는 호스트 툴.

---

## 1. 마일스톤 호 (초안 [추정])

```
[G0] 포트/포맷 결정 (설계 게이트)
        ▼
[G1] 시리얼 읽기 + monitor 포맷 파서
        ▼
[G2] buck 설정 입력 → UART5 송신
        ▼
[G3] 실시간 모니터 뷰 (헤더별 값 표시)
```

| 단계 | 달성 목표 | 완료 기준 | 상태 |
|------|-----------|-----------|------|
| **G0** | 어느 UART에 무엇을 붙일지 결정 | 설정 포트·모니터 포트 확정 | ✗ 미착수 ◄── ★ 설계 결정 선행 |
| **G1** | monitor 출력 파싱 | 통일 포맷 라인 → 구조화 데이터 | ✗ 미착수 |
| **G2** | buck 설정 송신 | GUI 입력 → 01 UART5 → 03 Monitor 반영 확인 | ✗ 미착수 |
| **G3** | 실시간 뷰 | 헤더별 값 갱신 표시 | ✗ 미착수 |

---

## 2. 설계 결정 — G0 (착수 전 호명 필요)

> **모름 — 사용자 결정 대기.** 추론으로 채우지 않음.

- **buck *설정*과 패킷 *모니터링*이 다른 포트일 수 있음.**
  - 설정(`buck <v>`)은 **01_RX_control UART5** (115200/8N1, [[uart_command_set]]).
  - 모니터링은 **03_TX_ble Monitor** (J-Link CDC UART/VCOM) 또는 **01의 `Monitor_Loop`** — 후자는 현재 주석처리 비활성([[status]], [[rx_control#메인-루프]]).
  - → GUI가 **포트 2개**를 동시에 다뤄야 할 수 있음. 어느 조합을 쓸지 결정 필요.
- **파싱 대상 포맷**: monitor 출력 통일 포맷 `[{eta-rx/eta-tx}:{spi/esb}:{in/out}]` + 헤더별 라인(0x50/0x51/0x52, `0xNN | 이름=값`). 산출 출처 = monitor-formatting 작업([[status]] 프로토콜·모니터 절, 커밋 `c9cf6a3`).
