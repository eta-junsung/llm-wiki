---

## P1 — 핀맵

### 시스템 연결 개요

```
[rx 제어보드]  ─── SPI Master ───▶  [02 UTO-NBK-52]  ─── ESB 무선 ───▶  [03 UTO-NBK-52]  ─── SPI Slave ───  [tx 제어보드]
  01_RX_control                        02_RX_esb                              03_TX_esb                          (시립대 준비)
```

### 핀맵 요약

SPI: 10ms 주기 | UART: 115200/8N1

| 신호 | rx/tx 제어보드 (STM32) | rx/tx esb (nRF52) |
|------|----------------------|-------------------|
| SPI CS (NSS) | PB12 | CN2 핀 3 |
| SPI SCK | PB13 | CN2 핀 8 |
| SPI MISO | PB14 | CN2 핀 5 |
| SPI MOSI | PB15 | CN2 핀 7 |
| UART TX | PC12 | — |
| UART RX | PD2 | — |

## P2 — 구동 확인 절차

펌웨어 플래시 완료 보드 기준.

### 1. 전원 투입 순서

순서 제약은 없으나 SPI master → slave 순 권장:

```
① TX_esb       (UTO-NBK-52)
② RX_esb       (UTO-NBK-52)
③ rx 제어보드  — PC GUI 연결 (USB)
④ tx 제어보드  — 필요 시
```

### 2. 정상 동작 판별

**LED (02·03 각각, UTO-NBK-52 기준 · 모두 active-HIGH)**

| LED | 정상 상태 |
|-----|----------|
| LED1 — System Ready | **상시 점등** (전원 인가 직후) |
| LED2 — SPI Comm St | **200ms 점멸** — SPI 링크 UP |
| LED3 — ESB Comm St | **상시 점등** — ESB UP / 소등 = DOWN |

**comm_st 케이스별 기대 동작**

| 상태 | LED2 (SPI) | LED3 (ESB) | GUI |
|------|-----------|-----------|-----|
| rx 제어보드·RX_esb·TX_esb 전부 ON | 점멸 | 점등 | SPI UP / ESB UP |
| rx 제어보드·RX_esb ON, TX_esb OFF | 점멸 | 꺼짐 | SPI UP / ESB DOWN |
| SPI 배선 단선 | 꺼짐 | 꺼짐 | SPI DOWN |

## P3 — PC GUI 사용법

### 실행

`rx_gui.exe` 실행

### 연결

1. rx 제어보드와 USB to TTL 연결
2. GUI 상단 Port 드롭다운에서 COM 포트 선택 → **Connect**
3. 상단 링크 표시 확인:
   - `Link: SPI UP  ESB UP` → 정상
   - `SPI DOWN` → SPI 배선 점검
   - `ESB DOWN` → TX_esb 전원 확인

### buck 지령 입력

> ⚠️ 현재 GUI에 표시되는 수치는 더미 데이터. 실제 센서값이 아님.

`Tx Buck Converter Vout Ref Set` 입력란에 값 입력 → **Send**

- 범위: 0.00 ~ 300.00 V
- 전달 경로: GUI → rx 제어보드 → SPI → ESB → TX_esb
- 확인: 0x51 패킷 `Tx_Buck_Vout_Ref` Raw·Physical 갱신
